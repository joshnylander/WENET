WENET Schema 1.2.2

The WENET project has defined a schema, called WENET Schema v1.2.2 to complement the WENET Protocol.

Schema: http://www.whatcomcounty.us/apps/wenet/schema/wenet/wenet_schema_1-2-2.xsd

Schema Guidelines:

In addition to the schema, implemetations MUST adhere to the following guidelines in deploying the schema.

  • use of ID/RefID for linking elements

Changes:

  1. 20060113: Moved min/max from sequence to elements.
  2. 20050811: Replaced "choice" and "all"s with "sequence" elements. Added min/max to new sequence elements. Fixed instanceRoot/rootNode min/max values.
  3. 20050714: Replaced wenet:supportedXPaths with wenet:instanceRoots which contains wenet:supportedXPaths and added wenet:namespace element. Standardized change log with that of the protocol.
  4. 20050601: Replaced wenet:instanceRoots with wenet:supportedXPaths and added the summary element to the results.
  5. 20050418: Released new schema version 1.2.1 as DRAFT with spelling changes and minor structure changes including changes to the CertificateType. Removed support for complex XPath by removing the statement element and related elements.
  6. 20041123: Added ID/RefID language. Marked document "FINAL". Updated date.
  7. 20041101: Released new schema version 1.2 with major and incompatible changes from version 1.1. Most notably, removed the requirement that instances be based on the GJXDM and removed references to the GJXDM.
  8. 20040518: Released new schema version 1.1 with major and incompatible changes from version 1.
  9. Major Changes:
    1. Made changes to support changes in the WENET Protocol.
    2. Made changes to make it more compatible with the GJXDM v3.0.
