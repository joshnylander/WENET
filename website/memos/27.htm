WENET Technical Policy v1.3

WENET Technical Policy v1.3:

  1. Services MUST refresh their combined authlist at least every 30 minutes.
  2. Services MUST conduct service discovery at least every 24 hours.
  3. The wenet:owner and wenet:userinfo elements MUST contain URLs that point to a wenet:instance document with a data element root of gjxdm:ContactInformation.
  4. A result list is started within the 90 second timeout window does not have to finish within the 90 seconds to be valid.
  5. XPath root schemas (Protocol 6.3) and thus all data returned from WENET, MUST be a subset of the Global JXDM v3.0.2.
  6. Import statements used in the root schemas (Protocol 6.3) MAY use http://www.whatcomcounty.us/apps/wenet/schema/.
  7. When not specified, the default namespace prefixes and coresponding namespaces for Global JXDM queries SHALL be those used by the Global JXDM Schema Subset Generation Tool (http://gjxdmtools.gtri.gatech.edu/ssgt/viewer?changeMode=SEARCH_MODE).
  8. Each service MUST support, at a minimum, the query of XPaths as defined in the current WENET Minimum XPaths.
  9. If an element derived from PersonType has alert code(s) then the attribute sensitivityText MUST be added with value "HAS ALERTS".
  10. If ROOT Person has alert codes then these alert code(s) should be returned as a space seperated list of alert codes in the j:PersonDescriptionText element.
  11. Each ROOT must provide a sourceIDText attribute as specified in the current WENET Minimum XPaths. This attribute MUST be unique to the ROOT on that service point and MUST allow retrieval of the same data element when queried for later. Thus an XPath query of /j:Person@sourceIDText=12345 MUST return the same person each time.
  12. When referencing another ROOT within a ROOT, the sourceIDText attribute identifing the referenced data SHOULD be included. When it is included ONLY the child root elements who are listed in the current WENET Minimum XPaths MUST be included. Other child root elements MAY be excluded with the understanding that the client can retrieve the full detail by quering the child root using the sourceIDText attribute.
  13. Services implementing the getdeleted method MUST keep track of deleted records going back at least 10 days.
